Anwendungslogik:
Als erstes wird der User dazu aufgefordert seine Größe und sein Alter Gewicht, Geschlecht und Aktivitätslevel einzugeben. Daraus wird dann mithilfe der Harris-Benedict-Formel der tägliche Kalorienbedarf berechnet. WIE BERECHNET MAN RESTLICHE MARKROS, RECHERCHIEREN
Die errechneten Zielwerte für den täglichen Bedarf an Kalorien, Kohlenhydraten, Fetten und Proteinen werden dann in einem Objekt gespeichert. 
Danach kann der Benutzer Rezepte suchen, indem er entweder nach Zutaten oder einem Konkreten Rezept sucht. Das Programm hat vorher die Rezept JSON Dateien eingelesen und in einem Array eingespeichert. Dann durchsucht es dieses mithilfe der String funktion "includes" oder "search". Das Resultat wird dem Nutzer nun präsentiert. Wenn er dies bestätigt, werden die Zutaten und deren Mengenangaben formatiert, sodass diese von der API empfangen und verarbeitet werden können. Nun werden sie mit dem Application Key und der Application ID an die API gesendet.  Die API schickt die Nährwertangaben der angefragten Zutaten zurück. 
Außerdem besteht noch die Möglichkeit einzelne Zutaten mit Mengenangaben direkt zu suchen. Damit wird genauso vorgegangen wie mit den Zutatenlisten aus Rezepten. Bei jeder weiteren Nährwertanfrage wird neu errechnet wie viel von dem Tagesbedarf erreicht wurde und das in dem Objekt gespeichert. Auf Wunsch des Nutzers wird das Objekt in eine JSON Datei gespeichert und kann ausgegeben werden.
